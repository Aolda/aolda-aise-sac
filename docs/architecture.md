# AISE OS Security Architecture

## 1. 개요 (Overview)
본 문서는 AISE (Aolda Infrastructure Security Enhancement) 프로젝트의 **OS 인프라 보안 조치**를 위한 Security as Code (SaC) 아키텍처를 정의합니다. 
Ansible을 기반으로 작동하며, CIS ubuntu benchmark와 KISA 클라우드 취약점
점검 가이드에 따라 **4가지 핵심 책임 영역**으로 모듈화하여 서버 보안설정 자동화 Ansible Playbook 개발합니다.

---

## 2. 아키텍처 및 디렉터리 구조 (Directory Structure)
OS 보안 조치를 담당하는 Role은 4가지 핵심 책임 영역으로 나눠 각각 독립적인 파일로 분리하여 구현합니다.

```text
aolda-aise-sac/                     
├── docs/                           # 프로젝트 운영 및 가이드 문서
│   ├── architecture.md             # 프로젝트 전체 아키텍처 명세서
│   └── git-convention.md           # Git 협업 및 브랜치 명명 규칙 가이드
├── group_vars/                     # 그룹별 공통 적용 환경 변수
├── inventory/                      # 대상 인프라 목록
├── playbooks/                      # 통합 실행 플레이북
├── .gitignore                      # Git 버전 관리 제외 항목
├── roles/                          # 각 담당자별 Role 작업 공간
└── README.md                       # 프로젝트 소개 및 실행 가이드
```

## 3. 핵심 책임 영역 (Core Responsibility Areas)

### 3.1. 계정 및 인증 통제 영역
비인가자의 시스템 접근을 원천 차단하고 인증 크리덴셜 정책을 강화하여 계정 탈취 위험을 최소화합니다.
*   **root 계정 원격 접속 제한:** `/etc/ssh/sshd_config` 내 `PermitRootLogin no`를 설정하여 직접적인 원격 로그인을 차단하고 책임 소재를 명확히 합니다.
*   **패스워드 정책 강화:** 패스워드 최대 사용 기간을 90일 이하로 제한하고, 패스워드 복잡도 규칙을 강제합니다.
*   **계정 잠금 임계값 설정:** 로그인 실패 누적 시 계정을 잠가 무차별 대입 공격을 차단합니다.

### 3.2️. 주요 설정 파일 및 디렉터리 권한 통제 영역
중요 설정 파일의 무단 훼손 및 정보 유출을 막기 위해 파일 시스템의 소유자와 권한을 엄격히 제어합니다.
*   **계정 및 패스워드 파일 보호:** `/etc/passwd` 파일은 소유자를 root로 하고 644 이하의 권한을 부여하며, `/etc/shadow` 파일은 400 권한으로 엄격히 제한하여 암호 해시 유출을 방지합니다.
*   **시스템 설정 파일 보호:** `/etc/hosts`, `/etc/services` 파일의 소유자를 root로 지정하고 644 이하의 권한을 부여하여 로컬 네임 및 포트 매핑 변조를 막습니다.
*   **우회 인증 차단:** 비밀번호 없이 시스템에 접근할 수 있게 하는 `$HOME/.rhosts` 및 `hosts.equiv` 파일을 찾아 제거합니다.
*   **취약한 권한 점검:** 비인가자의 권한 상승 및 변조 방지를 위해 `SUID`, `SGID`, `Sticky bit`가 설정된 취약 파일과 누구나 쓰기 가능한 `world-writable` 파일을 점검하고 조치합니다.

### 3️.3. 불필요한 서비스 및 데몬 통제 영역
사용하지 않거나 자체적인 취약점이 존재하는 서비스를 비활성화하여 외부 공격 표면(Attack Surface)을 최소화합니다.
*   **취약 서비스 비활성화:** 정보 노출 및 악용 위험이 있는 `Finger`, `Anonymous FTP`, `tftp`, `talk` 서비스를 중지합니다.
*   **원격 및 공유 서비스 통제:** 암호화되지 않은 `r 계열 서비스(rsh, rlogin 등)`와 불필요한 `NFS 서비스`, `automountd`, `RPC`, `NIS` 구동을 차단합니다.
*   **DoS 공격 방어:** 트래픽 증폭 공격에 악용될 수 있는 `Echo`, `Chargen` 등 DoS 취약 서비스를 비활성화합니다.

### 3.4. 시스템 유지보수 및 인프라 운영 영역
알려진 시스템 취약점을 선제적으로 제거하고, 침해사고 발생 시 정확한 로그 타임라인 확보를 위한 운영 환경을 유지합니다.
*   **시간 동기화 무결성:** `NTP/Chrony`를 통해 시스템 시간을 자동 동기화하여 로그 타임라인의 무결성을 확보합니다.
*   **로그 파일 보호:** `/etc/(r)syslog.conf` 설정 파일 및 cron 관련 파일의 소유자와 권한을 보호하여 감사 로그와 스케줄러 변조를 방지합니다.
*   **최신 보안 패치 적용:** OS 및 패키지에 최신 보안 패치와 벤더 권고 사항을 지속적으로 반영하여 취약점을 제거합니다.
